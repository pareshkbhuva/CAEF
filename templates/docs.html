API Reference

Documentation

The AgnosLogic API is a JSON REST API over HTTPS. Authenticate with Bearer tokens. All responses include latency metrics and calibrated verdicts.

# Base URL https://www.agnoslogic.com/v1

Authentication

Every request requires a Bearer token in the Authorization header. Get your API key from the dashboard after signing in with Google.

Authorization: Bearer agnoslogic_YOUR_API_KEY

Rate limits

Rate limits are enforced per API key based on your plan. Usage resets at midnight UTC.

PlanDailyMonthly
Explorer (free)15450
Builder50010,000
EnterpriseUnlimitedUnlimited

Exceeding your limit returns HTTP 429 with a JSON error. Check current usage via GET /v1/usage.

Errors

Errors follow standard HTTP status codes. The response body includes an error field with a human-readable message.

StatusMeaning
400Missing or invalid request parameters
401Missing or invalid API key
429Rate limit exceeded
502Inference backend unavailable

POST/v1/score

Score a piece of text for epistemic risk. Returns a verdict without generating new content. Use this to verify claims, check outputs from other LLMs, or analyze existing text.

Parameters

FieldTypeDescription
textrequiredstringText to analyze. Max 2000 characters.

Example

curl https://www.agnoslogic.com/v1/score \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"text": "Water boils at 100°C at sea level."}'
# Response { "verdict": "verified", "category": "LOW", "latency_ms": 87 }

POST/v1/ask

Generate an answer to a question with a built-in verdict. The model produces a response and scores its own output in a single forward pass.

Parameters

FieldTypeDescription
questionrequiredstringThe question to answer.
max_tokensintegerMax response length. Default: 200. Max: 512.

Example

import requests r = requests.post( "https://www.agnoslogic.com/v1/ask", headers={"Authorization": "Bearer YOUR_KEY"}, json={"question": "How does antibiotic resistance develop?"} ) print(r.json())
# Response { "response": "Antibiotic resistance develops through...", "verdict": "verified", "category": "LOW", "latency_ms": 342 }

POST/v1/compare

Compare two statements and identify which is more likely to be hallucinated. Useful for A/B testing LLM outputs or verifying paraphrases.

Parameters

FieldTypeDescription
text_arequiredstringFirst statement.
text_brequiredstringSecond statement.
# Response { "text_a": {"verdict": "verified"}, "text_b": {"verdict": "flagged"}, "more_risky": "B", "latency_ms": 156 }

GET/v1/usage

Check current rate limit usage for your API key.

# Response { "plan": "free", "daily": {"used": 3, "limit": 15}, "monthly": {"used": 12, "limit": 450} }

GET/v1/health

Public endpoint. No authentication required. Returns API status.

Verdict categories

Every response includes a verdict field derived from an internal calibrated threshold.

VerdictMeaning
verifiedThe model is confident in the response — appears factually grounded.
uncertainThe model is unsure — verify independently before relying on this.
flaggedThe model's hidden states suggest fabrication — likely contains errors.

Response fields

FieldTypeDescription
verdictstringOne of: verified, uncertain, flagged.
categorystringInternal category: LOW, MED, HIGH.
responsestringGenerated text (only on /ask).
latency_msnumberEnd-to-end latency in milliseconds.
